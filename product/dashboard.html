Product
What it does

ManyClaw lets you run multiple OpenClaw instances on one Mac. Each instance is named, isolated, and managed from a single window — its own memory, sessions, and config. Start one for dev, one for production, one for experiments. Switch between them instantly. No terminal, no port management, no config files.

Vision

The simplest possible way to run and manage multiple OpenClaw instances on a single machine. Built for people who are actively testing, building, or developing OpenClaw agents — and who need more than one running at the same time without the friction of manual profile setup, port juggling, and terminal tabs.

One window. Named instances in a sidebar. Click to start. Everything isolated. You should be able to go from zero to three running agents in under a minute.

Strategy

Built for OpenClaw power users — developers and agent builders who need more than one instance running at a time. Not aimed at general consumers.

Open source under GPL-3.0. Build in public, grow with the OpenClaw community. The tool that builds itself is the best demo.

Stay a shell. ManyClaw manages instances — it does not replace OpenClaw's own UI, auth, skills, or config. Keep the surface area small.

macOS first. Signed DMG on manyclaw.app. Homebrew cask next. Windows and Linux only if there's real demand.

No paid acquisition. Word of mouth within the community, blog content written by the team that uses the product.

Sandboxing will deepen over time — from native --profile isolation today, to Anthropic sandbox environments and macOS VM-per-instance as options. The UI stays the same regardless of what runs underneath.

ManyClaw is a manager, not tied to OpenClaw specifically. As the agent runtime space matures, support for other compatible runtimes is on the table.

Roadmap
Now
macOS — stable v1
Polish the core experience. Reliable instance management, no rough edges.
Auto-update
In-app updates via GitHub Releases. Users should never have to manually download a DMG again.
Next
Anthropic Sandbox support
Run instances in Anthropic-managed sandboxes instead of local profiles. Deeper isolation, no local resource contention.
Windows version
Electron supports it. Needs node-pty cross-compile and Windows code signing. Strong demand from the community.
macOS VM sandboxing
Each instance in a lightweight macOS VM. Maximum isolation for security-sensitive workflows.
Later
Multi-runtime support
Manage non-OpenClaw agent runtimes. ManyClaw becomes the universal agent manager.
Linux version
For server and developer workflows. Lower priority than Windows.
Homebrew cask
Standard macOS distribution channel for developers.
Instance templates
Pre-configured instance types — dev, production, test. One click to spin up a known-good config.
